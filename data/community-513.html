Instaweb

Posted by znerol on June 15, 2011 at 4:42pm

Instaweb is a drush command which starts a development webserver on localhost for the current drupal installation much like rails server or django-admin.py runserver.

Compatibility and Requirements

Instaweb runs on Linux and requires lighttpd and mod_magnet.

Installation

Issue the command drush pm-install instaweb wherever you like it. Probably the best choice is in ~/.drush. Refer to the drush README.txt for more choices.

Usage

$ drush help instaweb
Run a development webserver for this drupal instance
Options:
--bind                                    Bind the testserver to this ip
                                           address. Default: localhost
--port                                    Use specified TCP port. Default:
                                           3000
--server                                  Path to lighttpd binary. Default:
                                           /usr/sbin/lighttpd

Drupal Long Term Support (LTS)

Posted by johnbarclay on June 16, 2011 at 6:39am

Drupal Long Term Support (LTS) project facilitates long term support of Drupal versions through patches. Its goal is to both help support older versions of Drupal and generate interest in official long term support.

Priorities

  • Backport of security patches from supported Drupal core versions.
  • Backport of security patches for unsupported Drupal contrib modules.
  • Information about security issues with past Drupal Core and Contrib.

Structure

  • Module code base would contain patches.
  • Module code base might contain functionality to find or see if patches already applied.
  • Issue queue for known security issues.

Resources

  • groups.drupal.org LTS Group

When a Drupal core version such as 5.x is no longer supported an LTS branch can be started for that core version.

Ultimately long(er) term support of Drupal core versions will not work without great community support and infrastructure beyond a contrib module. This project is simply leveraging the Drupal project infrastructure for LTS, starting with Drupal 5, until such support has more community support and an appropriate infrasctructure.

Modes: A Super-Features Module

Posted by IceCreamYou on June 21, 2011 at 4:00pm

Modes allows creating mutually exclusive Features. The goal is to create site "modes" -- typically collections of features that together compose a certain type of site. Because only one Mode can be enabled at a time, this allows administrators to switch between different site Modes, changing the type of site they're on. For example, this module could allow switching from a blogging site to a microblogging site by just clicking a radio button.
