Media Actions

Posted by pfournier on November 14, 2007 at 9:47pm

This modules provides actions (as defined by the Actions module) to help manage media files. Currently, it implements storing video files from CCK nodes on blip.tv (uploading, updating and deleting the file).

Drupal 6 Status: we are waiting for CCK and emfield to work properly in Drupal 6 before finishing the port of Media Actions to D6. In the meantime, we started porting Media Actions to Actions 2.0, which is a backport of Drupal 6 actions into Drupal 5. You may want to try that; it is not fully tested yet. Get it from CVS HEAD.

This module was funded by OpenSourceCinema.org.

Usage

To use these actions, you need to create a CCK node type with two fields: one Video file field (provided by the CCK mediafield module), and one Embedded Video field (provided by the CCK emfield module).

Then, create and configure the Media Actions; you need to put your Video file field name as the Local Video Field Name, and your Embedded Video field name as the Embedded Video field.

When creating a node, the user should enter a filename in the Video file field. The file will be uploaded as usual on your Drupal server, but then, the media action will upload this file to blip, delete the local file and modify the Embedded Video field to point to the newly created blip video.

Google Highly Open Participation Contest (GHOP)

Posted by webchick on December 3, 2007 at 1:05am

This is a project to manage tasks for the Google Highly Open Participation Contest™.

Important Reminders and Deadlines

  • All tasks must be marked closed by midnight Pacific time on Feb. 4, 2008 for you to receive credit. This means that you need to submit your task before that so there is time for you to make corrections if necessary.
  • You must upload a final copy of your work for each task you have completed to the Google task tracker. See the How to get credit for your task Wiki page for more information.
  • You must subscribe to the ghop-announce list on Google Groups so that you receive important information from Google about how to claim your prizes, etc.
  • You must have your parents fill out the official Google parental consent form and send that back to Google to claim your prize(s).
  • If you haven't already, please read the Claiming your GHOP prizes post on groups.drupal.org and add your name to the list of ghop-announce subscribers once you have subscribed so that we know you are getting the important information you need to get.

CGI:IRC

Posted by frollo on December 19, 2007 at 7:15pm

This is a Drupal injection of the CGI:IRC php code found on cgiirc.org. This project was created for Drupal so users could access IRC chat behind a firewall. Most offices and other networks block common ports used for IRC and many other applications. This script is run from the server instead of the end user thus bypassing the issue with blocked ports. Input data is fully customizable from the back end if needed, and then passes through to the script which is running in an IFrame.
